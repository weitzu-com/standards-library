# 文件夹与文件整理命名规范

> v1.0 | 2026-06-15 | 丰田 5S + ISO 9001
> 适用范围：全部本地文件系统

---

## 0. 第一性原理

```
文件管理的本质 = 在正确的时间，以最小的成本，找到正确的文件。

混乱的根因 = 无分类标准 + 无命名规则 + 无生命周期管理 + 无定期清理
```

四条铁律推导出四层规范：

| 原理 | 规范层 |
|------|--------|
| 每个文件只属于一个类别 | **§1 目录分类** |
| 文件名即元数据，看名知义 | **§2 命名规则** |
| 文件有生老病死 | **§3 生命周期** |
| 积灰会自己回来 | **§4 维护规程** |

## 1. 目录分类（整顿 Seiton）

### 1.1 分类原则

```
一级目录 = 领域（不超过 10 个）
二级目录 = 子领域（按需，不超过 7 个）
三级目录 = 禁止（扁平优先）
```

### 1.2 编号体系

```
0X_分类名/

00 — 索引/标准/规范（元信息）
01~09 — 按业务领域排列
90 — 归档（休眠项目）
99 — 临时区（7 天内处理）
```

### 1.3 标准分类模板

```
项目根目录/
├── 00_索引与规范/       ← README, CLAUDE.md, 命名标准
├── 01_核心业务/          ← 按实际业务命名
├── 02_支撑业务/
├── ...
├── 90_归档/              ← 超 6 个月未活跃
└── 99_临时区/            ← 中转站
```

### 1.4 禁止事项

- ✗ 根目录散落文件（README/CLAUDE 除外）
- ✗ 空目录长期存在
- ✗ `.DS_Store`、`Thumbs.db` 残留
- ✗ 三级及以上目录嵌套
- ✗ 同类文件分散在多个目录

## 2. 命名规则（清洁 Seiketsu）

### 2.1 通用规则

```
[主体]_[描述]_[版本]_[日期].[扩展名]

规则：
- 分隔符统一使用下划线 `_`（禁止空格、短横混用）
- 中文优先（便于搜索），英文缩写次之
- 日期格式：YYYYMMDD 或 YYYY-MM-DD
- 版本号：v1, v2 或 年份
- 禁止特殊字符：/\:*?"<>|()[]{}!@#$%^&*
```

### 2.2 按文件类型细分

#### 文档类

```
[主题]_[文档类型]_[日期].md

示例：
  产品手册_GRP检测_v2_20260615.md
  会议纪要_周例会_2026-06-15.md
  工作日志_2026-06-15_网站部署.md
```

#### 标准/规范类

```
[组织]_[标准号]-[年份]_[名称].[扩展名]

示例：
  API_15HR-2016_4th_Edition.pdf
  GB_T_3854-2017_增强塑料巴柯尔硬度试验方法.pdf
  ISO_14692-1-2017.pdf
  SYT_6267-2018_高压玻璃纤维管线管.pdf
```

#### 图片/素材类

```
[主体]_[描述]_[序号].[扩展名]

示例：
  hero_banner_01.webp
  logo_primary.svg
  product_ats_testing_01.webp
```

#### 代码/配置类

```
[项目]_[模块]_[功能].[扩展名]

示例：
  leisa_Header.astro
  leisa_global.css
  vercel.json
```

#### 表格/数据类

```
[主体]_[数据范围]_[日期].csv

示例：
  客户清单_2026Q2_20260615.csv
  报价记录_2026-06.csv
```

### 2.3 版本管理

```
当前版：  文件名_v2.pdf        ← 最新
旧版归档： 00_过期版本/文件名_v1.pdf
```

旧版本不移入过期目录——超过 2 个历史版本时，只保留最近 1 个旧版。

### 2.4 禁止的命名模式

| 模式 | 问题 | 修正 |
|------|------|------|
| `文件名 (1).pdf` | 副本无标识 | 删除或重命名 |
| `文件名-最终版.pdf` | "最终"从来不是最终 | 用 v1/v2/v3 |
| `新建文件夹/` | 无意义 | 按分类命名 |
| `文件名_免费标准网_xxx.pdf` | 来源标记是噪音 | 删除来源后缀 |
| `文件名_OCR.pdf` | OCR 是技术标记 | 保留原始版本即可 |

## 3. 生命周期（整理 Seiri）

### 3.1 文件三态

```
活跃 → 休眠 → 死亡

活跃（<6个月）     → 留在工作目录
休眠（6~12个月）   → 移至 90_归档/
死亡（>12个月）    → 删除（确认无法律/审计需求后）
```

### 3.2 收件箱模式

```
99_临时区/  ← 所有新文件先放这里
     ↓ 7 天内分类
01~09 工作目录  →  90_归档  →  删除
```

### 3.3 清理触发条件

| 触发条件 | 动作 |
|----------|------|
| `.DS_Store` 出现 | 立即删除 + 添加 `.gitignore` |
| 发现重复文件 | MD5 对比 → 保留最优命名 → 删除其余 |
| 目录文件 > 50 个 | 检查是否需要拆分子目录 |
| 磁盘占用 > 80% | 全盘 5S 扫描 |

## 4. 维护规程（素养 Shitsuke）

### 4.1 日清 (Daily)

- 新增文件放入对应分类目录
- 临时文件移入 `99_临时区/`
- 下载文件 24h 内处理

### 4.2 周检 (Weekly)

- 清理 `99_临时区/`
- 检查根目录散落文件
- 删除 `.DS_Store`

### 4.3 月检 (Monthly)

- 重复文件扫描
- 休眠文件移入 `90_归档/`
- 更新 `README.md` 索引

### 4.4 年检 (Yearly)

- 全盘 5S 审计
- 过期文件清理
- 分类体系评估与优化

### 4.5 检查清单

```
□ 根目录仅含 README.md + 编号目录
□ 所有目录有编号前缀 (0X_)
□ 所有文件名符合命名规范
□ 无 (1)(2) 等副本后缀
□ 无 .DS_Store / Thumbs.db
□ 无空目录
□ 99_临时区 已清空（<7天）
□ README.md 已更新索引
```

## 5. 快速参考卡

### 新建文件时

```
1. 判领域 → 选 0X_ 目录
2. 按 §2 命名 → 保存
3. 更新 README.md（如是重要文件）
```

### 发现混乱时

```
1. 删除系统垃圾 (.DS_Store / Thumbs.db)
2. 合并同类目录
3. 扁平化子目录
4. 去重（MD5 对比）
5. 重命名（按 §2 规范）
6. 更新 README.md
```

### 目录编号速查

| 编号 | 用途 | 示例 |
|------|------|------|
| 00 | 元信息 | 索引、规范、CLAUDE.md |
| 01 | 核心 A | 产品、客户、合同 |
| 02 | 核心 B | 技术、研发、标准 |
| 03 | 支撑 A | 财务、人力、行政 |
| 90 | 归档 | 休眠项目 |
| 99 | 临时 | 中转站 |

## 附录 B：常见问题

**Q: 一个文件属于两个类别怎么办？**
A: 选「主用途」归类。另一个用途在 README.md 中标注交叉引用。

**Q: 旧版标准要保留吗？**
A: 移入 `00_过期版本/`。除非有法律/审计要求，否则不删除。

**Q: 中文还是英文命名？**
A: 中文优先（母语搜索最快）。国际标准号保留英文缩写（如 API、ISO、GB）。

**Q: 什么时候建子目录？**
A: 单目录超过 50 个文件时，按子类别拆分为二级目录。
