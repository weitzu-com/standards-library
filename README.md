## 第一性原理

```
标准索引的本质 = 在正确的时间，以最小的成本，找到正确的现行标准。

由此推演出：
  → 主目录只存现行标准（过期/撤销另册归档）
  → 一文件一行索引（四维定位：路径/等效/产品线/试验类型）
  → 异常高亮（缺增补/待核实 色标标记）
  → PDCA 闭环维护（年审计 + 季抽查）
```

## 命名规范

```
[组织]_[标准号]-[年份]_[名称]_[语言].扩展名

规则:
- 分隔符统一使用下划线 `_`
- 中文名称优先，国际标准号保留英文缩写
- 语言后缀: _CN / _EN / _RU（中文默认可省略）
- 过期版本归档至 00_过期版本/，原文件名不变
```

示例:
```
API_15HR-2016_高压玻璃钢管_CN.pdf
ASTM_D1599-2018_管道短时静水压试验_EN.pdf
GBT_29165.1-2022_石油天然气工业_玻璃纤维增强塑料管_第1部分-词汇、符号、应用及材料.pdf
SYT_6267-2018_高压玻璃纤维管线管.pdf
GOST_TU_2296-001-30372160-2016_RU.pdf
```

## 维护规程 (PDCA)

| 阶段 | 频率 | 动作 |
|------|------|------|
| P (计划) | 年度 | 每年12月全量版本审计，对照官网查新 |
| D (执行) | 即时 | 新标准按命名规范归入 0X_ 目录，同步更新 Excel 索引 |
| C (检查) | 季度 | 抽查：文件位置? 版本最新? 命名规范? 过期已归档? |
| A (改进) | 即时 | 发现过期→归档；发现缺增补→标记；发现新标准→采购 |
